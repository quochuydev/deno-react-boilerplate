Everything you need, nothing you don't

The electron-react-boilerplate experience, reimagined on the Deno runtime.

⚡

Faster Iteration: Hot Reloading

Edit your app and see changes instantly. deno desktop --hmr pairs with an esbuild watch build so the renderer refreshes without a full restart.

🧩

Scalable: Native TypeScript

Deno runs TypeScript with zero config. Shared channel types flow across the main process and renderer, so RPC calls are type-checked end to end.

📦

Performance: esbuild Bundling

The renderer is bundled and minified by esbuild — React included locally, no CDN. Tree-shaking and offline-ready builds come out of the box, ideal for kiosks.

🔒

Secure by Default

Deno's permission model means the app only gets the network and file access you grant it — a safer baseline than a full Node runtime.

🔄

Built-in Auto-Update

Deno.autoUpdate polls your release server and stages signed bsdiff patches to apply on next launch — wired up and ready.

🧪

Batteries Included

deno fmt, deno lint, and deno test replace ESLint, Prettier, and Jest — one toolchain, no extra dependencies.

From Electron to Deno

Same mental model as electron-react-boilerplate — mapped onto the Deno desktop runtime.

Electron concept Here (Deno + React)
Main process (Node) src/main/ — server, auto-update, RPC handlers
Preload / context bridge src/preload/bridge.ts → injected window.eva
IPC (ipcMain / ipcRenderer) HTTP-RPC routes /rpc/:channel + window.eva.invoke()
Renderer (React in Chromium) src/renderer/ — React in the webview
webpack (.erb/configs) deno.json tasks + esbuild
package.json / tsconfig.json deno.json
ESLint + Prettier + Jest deno lint / deno fmt / deno test

Get Started

Clone, then launch the native window. You'll need Deno 2.x. 

# clone the boilerplate
git clone https://github.com/quochuydev/deno-react-boilerplate
cd deno-react-boilerplate

# build the renderer + open the desktop window
deno task start

# or preview in a browser without the native window
deno task serve   # http://localhost:8000
